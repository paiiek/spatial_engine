## 3. 핵심 기술 스펙

- **플랫폼**: Linux Ubuntu 22.04 LTS, PREEMPT_RT 커널
- **오디오 I/O**: Digigram ALP-Dante (PCIe), JACK/PipeWire
- **블록 크기**: 64 frames @ 48 kHz
- **레이턴시 목표**: OSC 수신 → Dante PCIe 출력 p99 < 5 ms (stages 2–6)
- **공간화 알고리즘**: VBAP (Pulkki 1997), WFS, DBAP (Lossius 2009)
- **최대 오브젝트**: 16개 동시
- **리버브**: 16-line Hadamard FDN, denormal guard (±1e-20 DC offset)
- **바이노럴**: KEMAR SOFA (RWTH Aachen, IR_len=384, 64800 positions), 분할 컨볼루션
- **IPC**: OSC/UDP, 포트 9100(cmd)/9101(state), 10Hz 하트비트, 300ms 미스 게이트
- **UI**: PySide6 탑-다운 뷰, 드래그 120Hz, 매트릭스 뷰, 노이즈 패널

## 9. v0 통합 회로 보강 (2026-05-02)

### 배경
v0/v1/v1b 모듈이 모두 단위 테스트는 통과했으나 audit 결과
`SpatialEngine::audioBlock` 통합 경로가 `sine → VBAP → speaker` 만 결선되어
PerObjectChain / DBAP / WFS / FdnReverb / BinauralMonitor / Noise / Transport
가 **dead code** 상태였음. 본 절에서 통합 경로를 완성한 결과를 기록.

### 9.1 통합된 모듈 (engine entry → audio output 사이의 실 경로)

| AC | 모듈 | 통합 위치 (`SpatialEngine.cpp::audioBlock`) | 상태 |
|----|------|------------------------------------------|------|
| AC3 | `dsp::PerObjectChain` (EQ → DelayLine → DistanceGain → DistanceLPF → PropagationDelay → ReverbSend) | 액티브 객체별 `processSample` 루프 | ✅ |
| AC2 | `render::VBAPRenderer` / `DBAPRenderer` / `WFSRenderer` | per-algorithm scratch + 마스킹된 ObjectState span으로 dispatch | ✅ |
| AC4 | `reverb::FdnReverb` | 모노 send → wet → 출력 채널에 균등 분배 (`1/√n_spk`) | ✅ |
| AC6 | `output::BinauralMonitor` | 채널 `n_spk`/`n_spk+1` (가용 시), 첫 액티브 객체의 az/el 추적 | ✅ |
| 편의 | Noise generator (white/pink, per-channel) | xorshift32 + 1-pole LP, 출력 버스에 더함 | ✅ |
| 편의 | Transport play/stop | `transport_play_` atomic으로 모든 출력 게인 0/1 | ✅ |

### 9.2 새 OSC 명령 (UI → core)

| 주소 | 인수 | 동작 |
|------|------|------|
| `/obj/algo` | `,ii obj_id algo_int` | 0=VBAP, 1=WFS, 2=DBAP |
| `/obj/dsp` | `,iif obj_id param_id value` | param 0..3=EQ band dB, 4=delay ms, 5=k_hf, 6=reverb send |
| `/transport/play` | (없음) | 출력 게인 1.0 |
| `/transport/stop` | (없음) | 출력 게인 0.0 (모든 채널) |
| `/noise/{ch}/type` | `,s "white"\|"pink"` | 채널별 노이즈 타입 |
| `/noise/{ch}/gain` | `,f gain_db` | 채널별 노이즈 dB (≤-60 → 무음) |

### 9.3 UI 노출

| UI | 컨트롤 | 라우트 |
|----|--------|--------|
| PySide6 `ObjectInspector` | algorithm 콤보, EQ 4밴드, delay, HF, reverb send 슬라이더 | `OscClient.send_object_algo / send_object_dsp` |
| PySide6 `TransportPanel` | ▶ Play / ■ Stop | `OscClient.send_transport_play / send_transport_stop` |
| PySide6 `NoisePanel` | white/pink + dB 슬라이더 | (이미 있음) `send_noise_type / send_noise_gain` |

| WebGUI 디스패처 | obj_algo / obj_dsp / noise / transport (메시지 타입) | `osc_send_fn` 통해 9100 송신 |

### 9.4 RT 안전성 검증
- `chains_` 는 `std::vector` 힙 할당 (각 `DelayLine` ≈192 KB → 64 chain × 2 line = ~24 MB → 스택 불가)
- `prepareToPlay`에서 모든 scratch 사전 할당; `audioBlock`은 alloc-free
- `test_p1_rt_no_alloc` 회귀 통과 (RT-asserts 빌드)
