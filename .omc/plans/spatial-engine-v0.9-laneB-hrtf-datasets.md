## RALPLAN-DR Summary (for Architect/Critic alignment)

### Principles (3–5)
1. **Reuse, don't rebuild.** The engine already reads generic SOFA-derived `.speh` via `SofaBinReader`, builds a `KdTree3D`, and convolves per-object. Lane B adds a *catalog* + a *runtime swap*, nothing in the DSP kernel.
2. **All allocation + KdTree build on a non-audio thread; the audio thread does one acquire-load.** Mirror the v0.8 P1.1 *publish/consume protocol*: a 2-slot double buffer + single `std::atomic<int> active_sofa_slot_`, control publishes store-release, audio reads load-acquire once per block. **The double buffer covers `table_`/`tree_` ONLY** — the per-object B1 convolver banks self-heal (Principle 3); the B2 VS bank is a separate decision (see B-M2 scope split).
3. **The audio thread never touches the filesystem, never allocates a SOFA table, never builds a tree — but it DOES re-look-up HRIRs.** `setDirection()` runs ON the audio thread (`core/src/core/SpatialEngine.cpp:886`, inside `audioBlock()`), querying the active table/tree (`core/src/output_backend/BinauralMonitor.cpp:122`). So a SOFA switch is "prepare a fully-built inactive table+tree slot, flip one atomic." The B1 per-object HRIR then **self-heals**: the very next block, `setDirection()` re-looks-up against the new active slot and reloads via its existing 2-block crossfade (`core/src/output_backend/BinauralMonitor.cpp:122–168`). No worker re-derive, no extra alloc, no click-suppression work.
4. **Datasets are data, not code.** A new dataset is a JSON catalog entry + a fetched `.speh`, never a recompile. License gating lives in the catalog and the fetch script, not in C++.
5. **Auto-fetch only what the license permits.** Only CC-BY datasets are auto-downloadable; everything else requires explicit per-dataset user consent (already granted for SADIE-II / CIPIC / HUTUBS per plan §271 exception).

### Decision Drivers (top 3)
1. **RT-safety of the live swap** — the current `/sys/binaural_sofa` only stores a path string consumed at the *next* `prepareToPlay()` (`core/src/core/SpatialEngine.cpp:177`, `:393`); there is **no runtime hot-swap today**. The whole risk of Lane B is adding one without violating the alloc-free audio contract. Commands are drained *inside* `audioBlock()` (`core/src/core/SpatialEngine.cpp:459`) on the **audio thread**, AND `setDirection()`/`processBlockB2()` themselves run on the audio thread (`core/src/core/SpatialEngine.cpp:886`, `:871`). So the table/tree *load+build* must be punted to the ~1 Hz control tick (`core/src/bin/spatial_engine_core.cpp:628`), while the audio thread only ever does an acquire-load of the active slot index and reads the published table/tree data.
2. **KdTree build cost vs. swap cadence** — `KdTree3D::build()` allocates and runs O(N log N); for ~500-direction sets this is sub-millisecond, but it MUST run on the control tick (never audio) and MUST be bounded so a pathological dataset can't stall the control path. The ≥1-block cadence guard from P1.1 (`core/src/ambi/AmbiDecoder.h:16–25`) must hold.
3. **Footprint & demote-state coherence** — 2 *table* slots × largest dataset (distinct from convolver memory; see Risk #4), plus the B1/B2 runtime-demote machinery (`core/src/output_backend/BinauralMonitor.h:552–599`) must reset cleanly on a SOFA switch so a fresh dataset starts from a nominal monitoring state.

## Guardrails

**Must Have**
- Audio thread on a SOFA switch: **only** `active_sofa_slot_.load(acquire)` once per block, then it reads the published table/tree and (for B1) re-looks-up via the existing `setDirection()` self-heal. Zero alloc, zero file I/O, zero tree build on the audio thread.
- All `loadSpeh()` + `KdTree3D::build()` happen on the ~1 Hz control tick into the **inactive** table/tree slot, published store-release. **No worker re-derive of per-object convolvers** (would race the audio-thread `setDirection()` write).
- New `.speh` files are gitignored under `assets/hrtf/`; only the catalog JSON + docs are committed.
- Auto-fetch limited to CC-BY datasets; per-dataset license string in catalog.
- Demote state (`runtime_demoted_`, strikes, latches) reset on a successful SOFA switch.

**Must NOT Have**
- No HDF5 / libsofa runtime dependency (keep the pre-baked `.speh` pipeline).
- No second full `BinauralMonitor` instance (Option B rejected).
- No swap cadence faster than one audio block (preserve the P1.1 BINDING INVARIANT; the Relacy test must fail if this slack is removed).
- No committed dataset binaries (license + repo-size).
- No change to the `.speh` on-disk format or `SpehHeader`.

### B-M2 — Double-buffer the SOFA table/tree (the load-bearing milestone)

**Scope split — what "done" means (unambiguous):**
- **IN SCOPE:** `table_`/`tree_` are double-buffered and hot-swappable live. **B1 (Direct) per-object rendering swaps live and for free** via the existing audio-thread `setDirection()` self-heal. This satisfies acceptance §B.1.2 (runtime hot-swap) for the default B1 path.
- **B2 (AmbiVS) policy — DEFAULT: quiesce-on-swap WITH A MANDATORY SINGLE-WRITER HANDSHAKE.** The single-slot VS bank (`core/src/output_backend/BinauralMonitor.h:497,499`, read by `processBlockB2()` at `.cpp:874–880`, built only in `initializeB2()` at `.cpp:331–338`) is rebuilt from the new active tree on the control tick. **`effectiveMode() != AmbiVS` is NOT a safe gate on its own — it is a TOCTOU race.** `effective_mode_` is written by the **audio thread**, not just at init: `runThroughputProbe()` can **promote back to AmbiVS** (`core/src/output_backend/BinauralMonitor.cpp:428`, with no handshake) and the auto-demote writes Direct (`:553`). Between the control tick reading `effectiveMode() != AmbiVS` and its `loadInto()` on the live `vs_conv_*_`, the audio thread can flip to AmbiVS and start a B2 block that `process()`es the very convolvers the control tick is mid-`loadInto()`. `isRuntimeDemoted()` narrows but does NOT eliminate this (the non-demoted throughput-promotion path at `:426–429` re-enables B2 with no handshake).
  - **Required mechanism (MANDATORY while quiesce is the default):** add `std::atomic<bool> vs_rebuild_in_progress_{false}`. The control tick sets it **release-true BEFORE any `loadInto()` on `vs_conv_*_`**, performs the rebuild, then sets it **release-false**. The audio-thread B2 dispatch, at the **existing single per-block `effective_mode_` snapshot site** (the C1-fix snapshot — `observeAndArmXfade()` loads `effective_mode_` acquire at `core/src/output_backend/BinauralMonitor.cpp:755`, called once per block at `core/src/core/SpatialEngine.cpp:796`), **acquire-loads `vs_rebuild_in_progress_` AT THE SAME SNAPSHOT POINT** (one snapshot per block, no re-read) and, when true, **FORCES the effective branch to Direct for that block.** This converts the loose "rebuild only while not AmbiVS" into enforced mutual exclusion with a definite handoff block, analogous to the existing `front_idx`/`crossfade_active` discipline.
  - **Fallback (no TOCTOU):** double-buffer the VS bank with a new `std::atomic<int> active_vs_slot_` mirroring the per-object `front_idx` pattern (`core/src/output_backend/BinauralMonitor.h:466`). This option does **not** have the TOCTOU problem and is the cleaner answer if the listening-test Direct flash is rejected. **But as long as quiesce-on-swap is the DEFAULT, the `vs_rebuild_in_progress_` handshake is MANDATORY, not optional.**
  - **Full B2 live-swap is an acceptable follow-up (deferred), and B.1.2 is satisfied in-scope by B1.**

## ADR — Lane B HRTF Dataset Diversification

- **Decision:** Add a JSON-driven HRTF catalog and a **lock-free 2-slot double buffer over the SOFA `table_`/`tree_`**, in `BinauralMonitor`/`HrtfLookup`. The table/tree slot reuses the v0.8 P1.1 publish/consume *protocol*; B1 per-object rendering swaps live for free via the existing audio-thread `setDirection()` self-heal. **B2 VS bank: implemented as a SECOND 2-slot double buffer (`active_vs_slot_`, `vs_conv_/vs_hrir_` → `[2][24]`)** — NOT the originally-planned single-slot quiesce + `vs_rebuild_in_progress_` handshake. **그 handshake는 race-free가 아님이 구현 중 Relacy로 증명됨**(in-flight AmbiVS 블록의 snapshot이 flag를 stale `false`로 읽으면 control tick의 `loadInto()`와 overlap — force-Direct flag에는 "이미 진입한 reader가 drain됐는가" ack/quiescence wait가 없음). 따라서 정확성 위해 double-buffer fallback(plan §B-M2 line 107이 명시 승인)을 in-scope로 채택. 두 더블버퍼(table+VS)는 **explicit quiescence handshake**로 race-free 보장: audio가 `finalizeXfadeBlock`에서 last-consumed slot을 release-publish, control의 `waitInactiveSlotQuiescent()`가 덮어쓸 슬롯이 더 이상 audio의 last-consumed가 아닐 때까지 대기(P1.1 binding invariant의 robust fix, ~1Hz 타이밍 slack에 의존 안 함). Runtime selection via new OSC verb `/sys/binaural_sofa_select ,s`, applied on the ~1 Hz control tick (`core/src/bin/spatial_engine_core.cpp:628`), surfaced in the existing Lane A dashboard. Datasets are fetched offline (CC-BY auto, others on consent) and never committed.
- **Drivers:** (1) runtime hot-swap is a hard acceptance requirement (B.1.2) and does not exist today (current `/sys/binaural_sofa` is prepareToPlay-only); (2) the audio thread alloc-free / no-file-IO contract is non-negotiable, and `setDirection()`/`processBlockB2()` run ON the audio thread; (3) the P1.1 publish/consume protocol is proven + has a Relacy precedent, minimizing concurrency surface for the table/tree slot.
- **Alternatives considered:** Option B (re-run full `initialize()` behind a pointer swap) — rejected: doubles the entire convolver graph and creates a large lock-free surface for marginal reuse. Option C (catalog only, no live swap) — rejected: violates B.1.2. (For B2 specifically: full VS double-buffer with `active_vs_slot_` is the considered-but-deferred alternative to quiesce-on-swap.)
- **Why chosen:** Smallest correct delta that satisfies B.1.2 (for B1) while preserving the RT contract; reuses the audited P1.1 protocol for the table/tree and the existing B1 self-healing crossfade and the Lane A WebGUI shell verbatim.
- **Consequences:** Adds ~1 MB resident (2 table slots; convolver memory is separately bounded by `kOlaMaxIRLength` and NOT doubled). A new control-tick entry point invoked alongside `applyPendingAmbiDecoderChange` (`spatial_engine_core.cpp:628`). A SOFA switch resets demote state (intentional, documented). **The B2 VS bank is novel concurrency surface with no P1.1 analog** (single-slot, audio-read, no `front_idx`); under the default quiesce policy a B2-active swap briefly renders Direct until the next control tick completes the VS rebuild. **That rebuild is guarded by a mandatory `vs_rebuild_in_progress_` handshake** (control sets release-true before any `loadInto()`, audio acquire-loads it at its single per-block snapshot and force-Directs the block when true) — without it, the audio-thread B2 re-promotion path (`BinauralMonitor.cpp:428`) creates a TOCTOU read/write race on `vs_conv_*_`. The stale `BinauralMonitor.h:6` threading comment is corrected.
- **Follow-ups:** full B2 live-swap (double-buffer the VS bank with `active_vs_slot_`) if the Direct flash is objectionable; pre-build all catalog KdTrees at init to bound swap latency; per-listener personalization UI; Windows fetch-script parity; expose active-SOFA name on `/ws/metrics` for dashboard reflection.
